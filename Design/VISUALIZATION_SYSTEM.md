# RepoGalaxy 可视化系统设计

> 状态：设计中 | 最后更新：2026-03-13
> 核心概念：Apple Watch App Library + 数据可视化映射

---

## 1. 核心设计理念

将 GitHub 仓库以**气泡云 (Bubble Cloud)** 的形式呈现，每个仓库是一个圆形图标，通过多个视觉通道同时编码多维数据。

**参考**：Apple Watch App Library 的圆形网格 + 科学可视化的数据编码

---

## 2. 布局系统 (双维度排序)

### 2.1 网格布局原理

```
┌─────────────────────────────────────────────────────────────────────┐
│                                                                     │
│  布局规则：                                                          │
│  • 从上往下：Time 排序 (新 → 旧)                                     │
│  • 从左往右：Star 排序 (多 → 少)                                     │
│  • 圆形大小：由 Star 数决定 (半径非线性映射)                          │
│  • 自动排列：紧密气泡云算法 (Circle Packing)                          │
│                                                                     │
│  视觉效果：                                                          │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  ●━━━━●━━━●●                                                  │ │
│  │ ━●━━━━●━━━●━━━●                                               │ │
│  │  ━━━●━━━━●━━━●●                                               │ │
│  │ ━━●━━━●━━━━●━━━●●                                             │ │
│  │   ━━━━●━━━●━━━━●━━●                                           │ │
│  │  ━━━●━━━━●━━━●━━━●                                            │ │
│  │  (大小不同的圆形紧密排列，像气泡一样)                           │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  上方 = 最新提交                                                      │
│  下方 = 较旧                                                          │
│  左侧 = Star 多 (大圆)                                               │
│  右侧 = Star 少 (小圆)                                               │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### 2.2 排序维度详解

```
Y轴 (垂直)：时间维度
├── 度量：最后提交时间 (Last Pushed At)
├── 方向：上 → 下 (新 → 旧)
├── 分组：可按天/周/月聚类显示
└── 标签："今天", "昨天", "本周", "本月", "更早"

X轴 (水平)：流行度维度
├── 度量：Star 数量
├── 方向：左 → 右 (多 → 少)
├── 映射：非线性压缩 (见 3.1 节)
└── 视觉：左侧大圆密集，右侧小圆稀疏
```

### 2.3 动画入场

```
圆形入场动画：
- 从屏幕中心向外扩散
- 根据位置延迟 (stagger)
- 弹性缓动 (EaseOutBack)
- 时长：400-600ms

滚动时的视差：
- 大圆移动慢 (近景)
- 小圆移动快 (远景)
- 营造 2.5D 空间感
```

## 3. 圆形视觉编码系统

### 3.1 半径映射 (Star → 大小)

```
Star 数 → 半径 (非等比映射，压缩大数)

映射表：
┌───────────┬─────────┬──────────────┐
│ Star 范围  │ 半径(px)│ 等级名称     │
├───────────┼─────────┼──────────────┤
│ 0         │ 16      │ 尘埃         │
│ 1-100     │ 24      │ 微星         │
│ 100-200   │ 32      │ 小行星       │
│ 200-400   │ 40      │ 行星         │
│ 400-500   │ 48      │ 大行星       │
│ 500-1k    │ 56      │ 巨星         │
│ 1k-10k    │ 64      │ 超巨星       │
│ > 10k     │ 72      │ 星系         │
└───────────┴─────────┴──────────────┘

注意：
- 使用对数/阶梯映射，避免大仓库占据过多空间
- 最小保证 16px (可点击)
- 最大限制 72px (防止过度 dominance)
```

### 3.2 颜色编码 (语言 → 扇形饼图)

```
圆形内部显示扇形调色盘：

┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│  每个圆形是一个迷你饼图，展示仓库语言比例：                        │
│                                                                 │
│      ╭──────────╮                                               │
│     ╱  🔵      ╲        蓝色 = Rust (60%)                       │
│    │  🔵🟢⚪   │        绿色 = Python (30%)                     │
│     ╲  🟢      ╱        白色 = Other (10%)                      │
│      ╰──────────╯                                               │
│                                                                 │
│  语言颜色表 (GitHub 官方色)：                                     │
│  • Rust:    #dea584                                             │
│  • Python:  #3572A5                                             │
│  • JS:      #f1e05a                                             │
│  • TS:      #3178c6                                             │
│  • Go:      #00ADD8                                             │
│  • Java:    #b07219                                             │
│  • C++:     #f34b7d                                             │
│  • ... 其他语言                                                   │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 3.3 亮度编码 (新旧 → 明暗)

```
最后提交时间 → 亮度

映射规则：
┌────────────────┬───────────┬────────────────┐
│ 时间范围        │ 亮度值     │ 视觉描述       │
├────────────────┼───────────┼────────────────┤
│ < 24h          │ 100%      │ 最亮，闪耀     │
│ 1-7 天         │ 85%       │ 明亮           │
│ 1-4 周         │ 70%       │ 适中           │
│ 1-3 月         │ 55%       │ 较暗           │
│ > 3 月         │ 40%       │ 暗淡           │
└────────────────┴───────────┴────────────────┘

效果：
- 新仓库像"新星"一样明亮
- 旧仓库逐渐"暗淡"
- 一眼看出项目活跃度
```

### 3.4 闪烁频率编码 (活跃度 → 闪烁)

```
问题：闪烁频率和图形面积高频缩放映射到什么？

建议方案 A：Issue/PR 活跃度
┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│  闪烁频率 → 近期 Issue/PR 活动频率                               │
│                                                                 │
│  频率表：                                                        │
│  • 高频闪烁 (2Hz): 今天有新的 Issue/PR/Commit                   │
│  • 中频闪烁 (1Hz): 本周有活动                                    │
│  • 低频闪烁 (0.5Hz): 本月有活动                                  │
│  • 不闪烁: 长期无活动                                            │
│                                                                 │
│  视觉效果：                                                      │
│  活跃的仓库像"脉冲星"一样有节奏地闪烁                            │
│  一眼看出哪些项目正在积极开发                                    │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘

建议方案 B：Star 增速 (Hotness)
┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│  闪烁频率 → 近期 Star 增长速度                                   │
│                                                                 │
│  频率表：                                                        │
│  • 高频: 今天新增 > 10 stars                                    │
│  • 中频: 本周新增 > 50 stars                                    │
│  • 低频: 本月新增 > 100 stars                                   │
│  • 不闪烁: 增速平缓                                              │
│                                                                 │
│  视觉效果：                                                      │
│  热门新星会有明显的闪烁吸引注意力                                │
│  配合"热门"筛选器使用                                            │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘

建议方案 C：综合活跃度指数
┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│  活跃度 = w1*Commit频率 + w2*Issue活跃度 + w3*Star增速           │
│                                                                 │
│  这样更全面反映项目健康度                                        │
│  但需要更多 API 调用                                              │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 3.5 图形面积高频缩放编码 (???)

```
问题："图形面积高频缩放"具体指什么？

可能的理解 A：呼吸效果
┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│  圆形有节奏地缩放 (呼吸效果) → 映射到什么？                      │
│                                                                 │
│  建议：Fork 数量                                                  │
│  • Fork 代表项目被引用的次数                                     │
│  • Fork 多 = 影响力大 = 呼吸幅度大                               │
│                                                                 │
│  映射：                                                          │
│  • 0 Fork:       不呼吸                                          │
│  • 1-100 Fork:   轻微呼吸 (1.0 → 1.05)                           │
│  • 100-1k Fork:  明显呼吸 (1.0 → 1.1)                            │
│  • > 1k Fork:    强烈呼吸 (1.0 → 1.15) + 光晕效果                │
│                                                                 │
│  效果：                                                          │
│  有影响力的项目像"活的心脏"一样脉动                              │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘

可能的理解 B：悬停放大
┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│  Hover 时放大倍数 → 映射到什么？                                 │
│                                                                 │
│  建议：用户相关度                                                │
│  • 根据用户历史行为计算相关度分数                                │
│  • 相关度高 = Hover 时放大更多                                   │
│  • 暗示"这个项目很适合你"                                        │
│                                                                 │
│  映射：                                                          │
│  • 低相关:  放大 1.1x                                            │
│  • 中相关:  放大 1.2x                                            │
│  • 高相关:  放大 1.3x + 边框高亮                                 │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘

可能的理解 C：点击反馈
┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│  点击时的缩放反馈 → 固定动画                                     │
│  • 点击:  缩小到 0.9x (50ms)                                     │
│  • 释放:  弹性回到 1.0x (200ms)                                  │
│  • 这是通用的微交互，不映射特定数据                              │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 5. 筛选系统 (Pill 栏目)

### 5.1 二元筛选规则

```
Pill 栏目可配置两个筛选维度：

默认配置：
┌─────────────┬─────────────┐
│ 时间范围    │ 排序方式    │
└─────────────┴─────────────┘

可选维度 A (第一列)：
• 全部时间
• 今天
• 本周
• 本月
• 今年

可选维度 B (第二列)：
• 智能推荐 (默认)
• Star 最多
• 最新提交
• 最活跃
• 正在趋势

组合示例：
┌──────────┬──────────┐
│ 本周     │ 最活跃   │  → 本周最活跃的项目
└──────────┴──────────┘

┌──────────┬──────────┐
│ 本月     │ Star最多 │  → 本月获得最多 Star 的项目
└──────────┴──────────┘
```

### 5.2 高级筛选 (可选展开)

```
点击 [高级 ▼] 展开更多筛选：

☑️ Rust   ☑️ Python   ☑️ Go   ☐ Java   ...

[最小] 0 ─────●──────── 100k [最大]

主题标签：
☑️ cli   ☑️ web   ☐ ai   ☑️ tool   ...

排除已看：
☑️ 隐藏已浏览的仓库
```

## 6. 交互设计

### 6.1 悬停效果

```
鼠标悬停在圆形上：

• 圆形放大 1.1x (200ms, EaseOut)
• 显示 Tooltip：
  ┌─────────────────────────────┐
  │ 📛 repo-name                │
  │ 👤 owner                    │
  │ ⭐ 1.2k  🍴 234  👁️ 56      │
  │ 🏷️ rust • cli               │
  │ 🕐 更新于 2小时前            │
  └─────────────────────────────┘
• 其他圆形轻微暗淡 (聚焦效果)
```

### 6.2 点击效果

```
点击圆形：

• 圆形放大到 0.9x 再弹回 (点击反馈)
• 展开详情面板 (从右侧滑入)
• 或全屏展开 (圆形放大转场)
```

### 6.3 拖拽滚动

```
支持鼠标拖拽滚动：
• 按住左键拖拽：平移视图
• 滚轮：垂直滚动
• Shift+滚轮：水平滚动
• 双击：重置视图到默认位置
```

### 6.4 缩放视图

```
支持缩放：
• Ctrl+滚轮：缩放视图
• 缩小时：显示更多仓库
• 放大时：显示更大圆形，更多细节
• 双指捏合 (触控板)
```

## 7. 性能优化

### 7.1 渲染策略

```
虚拟滚动：
• 只渲染视口内的圆形
• 视口外 200px 预渲染
• 使用对象池复用圆形节点

Canvas 绘制：
• 大量圆形时使用 Canvas/SkiSharp
• GPU 加速
• 60fps 目标

细节层级 (LOD)：
• 缩小时：简化圆形，隐藏细节
• 放大时：显示完整信息
```

### 7.2 动画性能

```
闪烁/呼吸效果：
• 使用 CSS/Composition 动画
• 批量更新，避免单独重绘
• 共享动画时间轴

交互反馈：
• 使用硬件加速 transform
• will-change 提示
```
