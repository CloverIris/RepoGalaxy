## 1. 产品概述

### 1.1 产品定位

**RepoGalaxy** 是一款重新定义代码发现体验的 **GitHub 第三方桌面客户端**，将数据可视化、游戏化交互与实用工具功能融合，帮助开发者发现优质但冷门的开源项目。

**产品类别**: 开发者工具 + 数据可视化应用
**目标平台**: macOS (主要) / Windows / Linux
**商业模式**: 开源免费 + 可选捐赠

### 1.2 核心问题

> "GitHub 上那些拿到很多 star 的项目是怎么火起来的？我也开发了很多质量很高的仓库，但为什么往往都比较冷清？"

现有 GitHub 推荐机制的问题：
- Trending 只看增速不看质量
- 搜索依赖关键词匹配
- 优质冷门项目难以被发现
- 推荐算法缺乏个性化

### 1.3 解决方案

通过 **多维数据可视化** 和 **智能推荐算法**，让开发者：
- 一眼识别项目的活跃度、技术栈、影响力
- 通过视觉直观感受项目间的相似性
- 发现被算法埋没的优质项目
- 高效管理本地和远程仓库

### 1.4 产品愿景

**让每一个优质开源项目都被看见**

RepoGalaxy 不仅是一个 GitHub 浏览器，更是一个探索开源宇宙的望远镜。

## 2. 目标用户

### 2.1 主要用户画像

| 画像 | 描述 | 需求 |
|------|------|------|
| **探索者 Explorer** | 喜欢发现新技术的开发者 | 发现有趣的项目，了解技术趋势 |
| **学习者 Learner** | 想学习特定技术的初学者 | 找到优质的教程和示例项目 |
| **贡献者 Contributor** | 想参与开源的贡献者 | 发现需要帮助的冷门项目 |
| **建设者 Builder** | 寻找工具/库的开发者 | 快速找到适合的技术方案 |
| **管理者 Maintainer** | 维护多个项目的开发者 | 管理本地仓库，跟踪依赖 |

### 2.2 用户场景

**场景 1: 发现新技术**
> 小明想了解 Rust 异步编程，打开 RepoGalaxy，筛选 Rust + Async，通过气泡云一眼看出哪些项目最近活跃、社区健康。

**场景 2: 寻找工具**
> 小红需要一个 CLI 工具，浏览探索页面，发现一个小而美的项目（几百 stars），但最近更新频繁，Fork 数也不错，决定尝试。

**场景 3: 管理本地项目**
> 小刚有几十个本地仓库，RepoGalaxy 自动扫描并显示状态，一键 Pull 更新所有项目。

**场景 4: 批量收藏**
> 小美通过拖拽聚类找到 10 个相关的机器学习项目，批量 Star 并收藏到特定列表。

## 3. 核心设计理念

### 3.1 设计原则

| 原则 | 描述 |
|------|------|
| **数据可视化优先** | 通过视觉通道同时呈现多维数据，而非列表展示 |
| **游戏化交互** | 拖拽、摇晃、聚类等有趣交互，降低探索成本 |
| **沉浸式体验** | 星空背景贯穿始终，营造探索宇宙的感觉 |
| **渐进式复杂度** | 默认简单，高级功能可选，不强迫用户学习 |
| **性能优先** | 流畅的 60fps 体验，后台加载不阻塞 |

### 3.2 核心隐喻

**开源宇宙 (Open Source Universe)**

| 概念 | 现实对应 | 视觉表现 |
|------|----------|----------|
| 星球 | GitHub 仓库 | 圆形气泡 |
| 星系 | 技术领域聚类 | 大气泡包含小气泡 |
| 星光 | 项目活跃度 | 亮度、闪烁 |
| 引力 | 项目关联性 | 拖拽聚类 |
| 探索 | 发现新项目 | 漂浮、筛选、浏览 |

## 5. 交互设计规范

### 5.1 导航结构

```
左侧导航栏 (固定 72px)
├── 主页/探索 (默认选中)
├── 收藏
├── 仓库 (GitHub)
├── 本地
└── 设置 (底部固定)

主页内 Pill 筛选栏
├── 时间范围: 全部/今天/本周/本月/今年
└── 排序方式: 推荐/热门/Trending/最活跃/随机
```

### 5.2 核心交互流程

#### 5.2.1 气泡云浏览

```
用户动作              系统响应
─────────────────────────────────────────
启动应用       →     显示缓存气泡 + 星空背景
                    后台加载新数据
                    
鼠标 Hover     →     气泡停下
                    放大 1.05x
                    显示 Tooltip
                    其他气泡变暗
                    
鼠标离开       →     恢复漂浮
                    恢复亮度

左键点击       →     按下反馈 (0.95x)
                    展开详情卡片 (变形动画)
                    背景模糊 + 其他气泡变暗
                    
点击详情卡     →     全屏展开仓库视图
                    导航栏收缩
                    加载完整信息
                    
ESC/返回       →     退回气泡云
```

#### 5.2.2 拖拽聚类

```
用户动作              系统响应
─────────────────────────────────────────
左键拖拽气泡   →     气泡跟随鼠标
                    放大 1.1x
                    其他气泡变暗
                    
摇晃鼠标       →     检测摇晃动作
                    气泡发光 + 颜色偏移
                    显示"摇晃中..."
                    
松开鼠标       →     触发相似项目检索
                    同类小气泡被吸引 (飞入动画)
                    融合成大气泡 (1.5-2.5x)
                    显示 "+N" 计数
                    
点击大气泡     →     展开多选预览面板
                    网格显示子项目
                    支持勾选/全选
                    批量操作按钮
                    
长按 3 秒      →     倒计时动画
                    边框颜色变化 (黄→橙→红)
                    内部抖动
                    显示倒计时
                    
达到 3 秒      →     破裂动画
                    粒子效果
                    小气泡四散弹射
                    恢复各自漂浮
```

|------|------|--------|--------|

## 7. 数据与缓存策略

### 7.1 数据源

| 源 | 类型 | 频率 | 优先级 |
|----|------|------|--------|
| GitHub Trending | API | 每小时 | 高 |
| GitHub Search | API | 每日 | 中 |
| 个性化推荐 | 计算 | 实时 | 最高 |
| 随机发现 | 抽样 | 每次启动 | 低 |

### 7.2 缓存策略

```
缓存位置:
- macOS: ~/Library/Caches/RepoGalaxy
- Windows: %LOCALAPPDATA%/RepoGalaxy/Cache
- Linux: ~/.cache/RepoGalaxy

缓存限制:
- 默认最大: 2GB
- 自动清理: 超过 80% 时清理旧数据
- 手动清理: 设置中提供清理按钮

缓存内容:
- API 响应 (JSON)
- 仓库封面图
- 用户头像
- 推荐结果

清理策略:
- LRU (最近最少使用)
- 保留收藏的仓库数据
- 保留用户设置
```

### 7.3 懒加载

```
启动流程:
1. 显示缓存数据 (< 100ms)
2. 启动后台同步
3. 数据源 A (Trending) - 优先
4. 数据源 B (Search) - 其次
5. 数据源 C (个性化) - 按需
6. 数据更新时平滑过渡
```
