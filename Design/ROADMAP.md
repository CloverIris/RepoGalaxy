## 🎯 版本规划

### v0.1.0 - MVP 核心验证 (预计 4-6 周)
**目标**：验证数据模型和 GitHub API 集成可行性，搭建 Avalonia + FluentAvalonia 基础框架

#### 核心功能
- [ ] **基础设施**
  - [ ] 项目结构搭建 (Avalonia + .NET 8)
  - [ ] FluentAvalonia 主题配置
  - [ ] DI 容器配置
  - [ ] 日志系统 (Serilog)
  - [ ] SQLite 数据库初始化

- [ ] **GitHub 集成**
  - [ ] OAuth Device Flow 登录
  - [ ] Token 安全存储 (macOS Keychain / Windows DPAPI / Linux Secret Service)
  - [ ] 基础 API 封装 (Octokit)
  - [ ] Rate Limit 处理

- [ ] **数据模型**
  - [ ] Repository 实体设计
  - [ ] EF Core 配置
  - [ ] 基础 CRUD 操作
  - [ ] 本地缓存机制

- [ ] **最小可用 UI (Fluent Design)**
  - [ ] 主窗口框架 (NavigationView)
  - [ ] Mica/Acrylic 材质背景
  - [ ] 简单的仓库列表页 (ListBox + 自定义 ItemTemplate)
  - [ ] 搜索功能
  - [ ] 设置页 (Expander 风格)

#### 交付标准
- 能登录 GitHub 账号
- 能搜索和浏览仓库
- 数据能本地缓存
- 有基本的 Fluent Design 外观

## 🔄 迭代原则

1. **数据优先**：先跑通数据流，再优化 UI
2. **小步快跑**：每个版本都有可演示的功能
3. **用户反馈**：早期版本收集反馈调整方向
4. **技术债务**：每个版本预留 20% 时间还技术债
5. **跨平台意识**：功能开发时考虑三平台兼容性
